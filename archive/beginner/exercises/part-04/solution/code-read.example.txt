## Script

```bash
#!/usr/bin/env bash
set -euo pipefail

for f in *.log; do
  if [ ! -e "$f" ]; then
    echo "No .log files here." >&2
    exit 0
  fi
  lines=$(wc -l < "$f")
  bytes=$(wc -c < "$f")
  echo "$f: $lines lines, $bytes bytes"
done
```

## Explanation

This script walks every file ending in `.log` in the current directory and prints a one-line size summary for each one.

The two lines at the top tell Bash to be strict: `set -euo pipefail` means stop on the first error, treat undefined variables as errors, and fail a pipeline if any step fails.

Inside the loop, `wc -l < "$f"` and `wc -c < "$f"` use input redirection so the filename is not echoed alongside the count, which keeps the final `echo` clean.

## Follow-up

> What does each flag in `set -euo pipefail` actually catch?

- `-e`: exit immediately if any command returns a non-zero status.
- `-u`: treat references to unset variables as errors and exit.
- `-o pipefail`: a pipeline returns the rightmost non-zero exit, not just the last command's exit.

Together they turn "silently broken" into "stop loudly".
